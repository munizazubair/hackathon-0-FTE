## Draft Response Guidelines

### Standard Opening Phrases

**Professional (Muniya)**:
- "Thanks for reaching out!"
- "I appreciate you following up on this."
- "Great to hear from you!"

**Casual (Muniza)**:
- "Hey!"
- "Hi there!"
- "Thanks for the message!"

### Standard Closing Phrases

**Professional**:
- "Let me know if you need anything else."
- "Looking forward to connecting."
- "Feel free to reach out with any questions."

**Casual**:
- "Let me know!"
- "Talk soon!"
- "Cheers!"

### Action-Oriented Language

Always include:
- Clear next steps
- Specific timeframes
- Confirmation of what was requested

**Example**:
> "I'll send the report by end of day Thursday. I'll include the Q4 metrics and the revised forecast. Let me know if you need anything else in the meantime!"

## Workflow Rules

### When Email Arrives:

1. **Categorize** by sender
2. **Assess priority** using matrix
3. **Check for action items**:
   - Meeting request → Check calendar
   - File request → Locate file
   - Question → Research answer
   - FYI → Note and archive

### Moving Between Folders:

- **Keep in Inbox**: Requires your action or decision
- **Move to Needs_Action**: Has clear action items, waiting on external factors
- **Move to Done**: Completed, responded, or archived

### Auto-Archive Triggers:

- Test emails from known senders
- Marketing/newsletter content (after review)
- Google security alerts (after verification)
- Emails older than 7 days in Done folder

---

## Special Instructions

### Meeting Invitations:

- Always confirm timezone (EST/PST/UTC)
- Offer 2-3 alternative times if primary doesn't work
- Create calendar event immediately upon confirmation
- Set reminder 24 hours before

### File/Document Requests:

- Acknowledge within 1 hour
- Set delivery deadline (within 24 hours unless urgent)
- Verify file exists before confirming
- Follow up if file wasn't sent within promised timeframe

### Technical/Project Discussions:

- Be specific and detailed
- Use bullet points for clarity
- Include relevant links or references
- Confirm understanding of requirements

## Red Flags (Manual Review Required)

Flag for immediate manual review if:

- Unexpected security alerts
- Urgent requests from new senders
- Large file attachments from unknown sources
- Payment or financial information requests
- Unusual language patterns from known contacts
- Requests to bypass normal procedures

## Keyword Detection Rules

### Upgrade to URGENT if subject/content contains:

**Time-sensitive**:
- "urgent", "ASAP", "asap"
- "deadline", "due today", "due tomorrow"
- "immediate", "right away", "now"
- "critical", "emergency", "high priority"

**Meeting/Schedule**:
- "meeting today", "call in 30 minutes"
- "confirm ASAP", "need response today"

### Identify as MEETING REQUEST if contains:

- "meeting", "schedule", "calendar"
- "coffee chat", "call", "zoom"
- "available", "free time", "book time"
- Dates/times: "Thursday at 2 PM", "next week"

### Identify as FILE REQUEST if contains:

- "send me", "need the file", "attach"
- "document", "PDF", "report", "presentation"
- "can you share", "forward the"

### Identify as NEWSLETTER if contains:

- "unsubscribe", "view in browser"
- "weekly digest", "newsletter"
- "marketing" (from known sender)
